# Future Improvements

## Purpose

This document contains ideas that may be added after Version 1 is complete.

These features should NOT be implemented during MVP development.

The MVP should remain focused on simple wildlife reporting.

---

# Version 2 Possibilities

## AI Species Suggestions

Users upload a photo.

The system suggests possible species.

Examples:

* "This may be a Bald Eagle."
* "This may be a Mule Deer."

The user must always confirm the species.

Accuracy is more important than automation.

---

## Offline Mode

Allow observations to be recorded when no internet connection exists.

Observations would sync when connectivity returns.

Useful for:

* Backcountry users
* Hikers
* Campers
* Hunters

---

## User Accounts

Potential features:

* Observation history
* Saved observations
* Personal statistics

Not required for Version 1.

---

## Administrative Dashboard

Allow administrators to:

* Review observations
* View maps
* Export data
* Monitor usage

---

## Data Export

Allow export to:

* CSV
* Excel
* PDF

Potential future provincial submission formats.

---

## Rare Species Alerts

Flag observations involving:

* Species at risk
* Rare ecosystems
* Sensitive habitats

May require expert review workflows.

---

## Provincial API Integration

Investigate whether provincial systems support:

* Direct submission
* Structured data exchange
* API integration

Do not assume an API exists.

Must be validated through stakeholder discussions.

---

## Citizen Science Integration

Potential integrations:

* iNaturalist
* eBird

Only investigate after MVP completion.

---

# Guiding Principle

Version 1 must remain simple.

Every future feature should be evaluated against the project's core goal:

Reduce friction between wildlife observation and wildlife reporting.
